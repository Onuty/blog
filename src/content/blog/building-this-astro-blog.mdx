I wanted this blog to feel fast and focused, so I moved from a traditional dynamic stack to **Astro + MDX + Content Collections**.

## Why this stack

- Static-first output for speed and reliability
- Git-backed content for transparent revision history
- Lightweight admin workflow through Decap CMS at `/admin`
- Discussion powered by giscus without ad-heavy scripts

## Publishing flow

1. Draft or edit from the CMS dashboard.
2. Save content as MDX files in the repo.
3. Deploy automatically through Cloudflare Pages.

```ts
const publishingFlow = ["Decap CMS", "Git commit", "Cloudflare Pages build"];
```

That workflow keeps the writing process simple while preserving full ownership of content.
